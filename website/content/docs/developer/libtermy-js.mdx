title: libtermy.js
description: First-party WASM terminal renderer for the browser — canvas2d / WebGL2 / headless backends, keyboard / selection / scrollback / OSC8 / mouse protocol in a single pass.
---

`libtermy.js` is the npm package for browser embedders. As of `0.2.0` it ships
a full first-party renderer: the WASM core parses bytes once and the canvas2d
or WebGL2 backend draws cells, handles input, manages selection, runs
scrollback, detects URLs / OSC8 hyperlinks, encodes mouse reports, and
exposes lifecycle events through a single `TermyRenderer` object.

The legacy `attachTermyToXterm` xterm.js adapter still exists for back-compat
but is deprecated — it double-parses every byte. See
[Migration](#migration-from-xtermjs-to-first-party-rendering) below.

## Install

```sh
npm install libtermy.js
```

No peer dependencies. `@xterm/xterm` is only needed for the deprecated xterm
bridge.

## Quick Start

```ts
import { loadTermy, createTermyRenderer } from 'libtermy.js'

const termy = await loadTermy()
const host = document.querySelector('#terminal') as HTMLElement
const renderer = createTermyRenderer(host, {
  termy,
  configContents: 'theme = nord\nfont_family = "JetBrains Mono"\nfont_size = 14\n',
})

const socket = new WebSocket('/pty')
socket.binaryType = 'arraybuffer'

renderer.onInput((data) => socket.send(data))
socket.addEventListener('message', (event) => {
  renderer.write(new Uint8Array(event.data))
})
```

`renderer.onInput` yields a `Uint8Array` (typed keystrokes + responses). Use a
`TextDecoder` if your transport requires strings.

`renderer.write(bytes)` feeds output into the WASM core and schedules a paint.
Terminal-response bytes (device-status queries, etc.) are forwarded back
through `onInput` automatically, so you do not need to inspect `responses`
manually.

## API Overview

### `loadTermy(options?)`

Loads the embedded WASM bundle and returns a `LoadedTermy` helper.

```ts
const termy = await loadTermy()
const term = termy.createRenderer(host, { configContents: 'theme = nord\n' })
```

The npm package embeds the WASM bytes into `dist/index.js`, so apps do not
need to copy `.wasm` files, configure a public asset path, or pass a
`wasmUrl`. Override with `moduleUrl` / `wasmUrl` only when hosting the
wasm-bindgen assets yourself.

### `createTermyRenderer(host, options)`

Creates a renderer. `host` is the `HTMLElement` to paint into — pass `null`
to construct a headless renderer that only holds a virtual buffer.

Key options:

| Option | Default | Notes |
| --- | --- | --- |
| `termy` | required | result of `await loadTermy()` |
| `core` | — | pre-built `TermyCore` (advanced) |
| `backend` | `'auto'` | `auto` / `canvas2d` / `webgl` / `webgpu` / `headless` |
| `workerized` | `false` | run the WASM core in a Web Worker |
| `cols`, `rows` | `80`, `24` | initial grid |
| `cellWidth`, `cellHeight` | `9`, `18` | initial cell metrics |
| `scrollback` | `5000` | scrollback ring size |
| `configContents` | — | Termy config text |
| `renderConfig` | — | pre-resolved render config |
| `fontFamily`, `fontSize`, `lineHeight` | from config | font overrides |
| `cursorBlink` | from config | toggle blink timer |
| `bellSound` | `'none'` | `none` / `visual` / `audio` |
| `rendererFps` | `60` | paint throttle |
| `macOptionIsMeta` | `false` | Alt-as-Meta on macOS keystrokes |
| `wordSeparator` | xterm default | string of word-boundary chars |
| `allowTransparency` | `true` | alpha blend with host bg |

### `TermyRenderer` shape

```ts
interface TermyRenderer {
  core: TermyCore
  backend: TermyRendererBackend
  cols: number
  rows: number

  write(data: string | Uint8Array): TermyFeedResult
  resize(cols?: number, rows?: number): void
  fit(): { cols: number; rows: number }
  focus(): void
  blur(): void

  scrollToBottom(): void
  scrollLines(amount: number): void

  getSelection(): string
  clearSelection(): void
  copy(): Promise<void>
  paste(text: string): void

  snapshot(): TermyFrame
  serialize(): Uint8Array

  search(query: string, options?: SearchOptions): TermySearchMatch[]
  searchAndHighlight(query: string, options?: SearchOptions): TermySearchMatch[]
  findNextMatch(): TermySearchMatch | null
  findPreviousMatch(): TermySearchMatch | null
  clearSearchHighlight(): void

  onInput(cb: (data: Uint8Array) => void): Disposable
  onResize(cb: (size: ResizePayload) => void): Disposable
  onSelectionChange(cb: (sel: SelectionPayload) => void): Disposable
  onLink(cb: (link: LinkPayload) => void): Disposable
  onTitle(cb: (title: string) => void): Disposable
  onWorkingDirectory(cb: (uri: string) => void): Disposable
  onProgress(cb: (payload: ProgressPayload) => void): Disposable
  onBell(cb: () => void): Disposable
  onClipboardStore(cb: (text: string) => void): Disposable

  dispose(): void
}
```

## Backends

| Backend | Use for | Notes |
| --- | --- | --- |
| `auto` | default browser apps | resolves to `canvas2d` |
| `canvas2d` | broadest browser support | glyph cache + per-cell damage diff |
| `webgl` / `webgpu` | high cell churn / large grids | LRU atlas, instanced quads, alpha-only glyphs w/ per-instance fg color, falls back to canvas2d if WebGL2 unavailable |
| `headless` | tests, SSR, AI agents | no DOM, only `snapshot()` / `serialize()` / `search()` |

Both DOM backends share the same interface. Switch is a single string change.

```ts
const renderer = createTermyRenderer(host, {
  termy,
  backend: 'webgl',
})
```

## Selection

Mouse drag selects cells. Click count escalates the mode:

- single click → cell mode (line-wrap selection)
- double click → word mode (word boundary expansion)
- triple click → line mode (full lines)
- Alt+drag → rectangular block selection

Selections are stored in **absolute buffer coordinates** so they survive
scrolling. When the anchor row is evicted from scrollback the selection
clears automatically.

```ts
renderer.onSelectionChange(({ active, startRow, startCol, endRow, endCol, text }) => {
  if (active) console.log('selection:', text)
})
```

Shift+click extends the existing selection from its anchor instead of starting
a new one.

## Search

`search(query)` returns matches synchronously. `searchAndHighlight(query)`
also paints them as an alpha overlay; the active match is outlined in the
theme foreground color. `findNextMatch()` / `findPreviousMatch()` cycle.

```ts
const matches = renderer.searchAndHighlight('TODO')
console.log(`${matches.length} hits`)
renderer.findNextMatch()
renderer.findPreviousMatch()
renderer.clearSearchHighlight()
```

## Links

Two link sources are merged into the renderer's link table:

- **Regex** detection of `https://` / `http://` / `ftp://` / `file://` URLs.
- **OSC 8** hyperlinks emitted by the terminal app (`\x1b]8;;<uri>\x1b\\`).

OSC 8 links are always underlined in the theme foreground color. Regex links
underline only on hover with the platform meta key held. Modifier+click on
either fires `onLink`:

```ts
renderer.onLink(({ uri, modifiers }) => {
  if (modifiers.meta || modifiers.control) {
    window.open(uri, '_blank', 'noopener')
  }
})
```

## Keyboard

Internally `dom-input.ts` translates DOM `KeyboardEvent` into a `Keystroke`
record, then `encodeKeystroke` produces the byte sequence the terminal
expects. The encoder supports:

- The legacy xterm-compat table: control combos, F1–F12, arrow keys
  (DECCKM-aware), Home/End/PgUp/PgDn, modifier-special shortcuts
  (Cmd+arrow word jumps on macOS, etc.).
- The full **Kitty enhanced keyboard protocol** when the application opts in
  via DECSET (`disambiguateEscapeCodes`, `reportEventTypes`,
  `reportAlternateKeys`, `reportAllKeysAsEsc`, `reportAssociatedText`).
- IME composition through a hidden contentEditable (`compositionend` fires
  the input bytes).
- Bracketed paste (`DECSET ?2004`) — pasted clipboard text is wrapped in
  `\x1b[200~` … `\x1b[201~` automatically when the mode is active.

Pass `macOptionIsMeta: true` to forward macOS Option-prefixed keystrokes as
ESC-prefixed bytes instead of letting the OS produce alt-graph characters.

## Mouse Protocol

`CSI ? 1000 / 1002 / 1003 / 1006 / 1015 / 1016 h/l` toggles xterm-compatible
mouse reporting. When the mode is active the backend encodes pointer events
through `core.encodeMouseReport(...)` and emits the bytes via `onInput`
instead of starting a selection drag. Shift held during a click overrides the
protocol and forces selection (matches xterm convention).

Supported encodings: **Legacy / X10**, **SGR**, **UTF-8**, **SGR-pixel**
(currently encodes cell coords until pixel coords are wired).

```ts
// inside the PTY stream
write('\x1b[?1006h')   // request SGR encoding
write('\x1b[?1002h')   // request button-event tracking
```

## Lifecycle Events

```ts
renderer.onTitle((title) => (document.title = title))
renderer.onWorkingDirectory((uri) => console.log('cwd:', uri))
renderer.onProgress(({ state, value }) => /* ConEmu OSC 9;4 — taskbar progress */ {})
renderer.onBell(() => /* tab badge flash, etc. */ {})
renderer.onClipboardStore((text) => /* OSC 52 — terminal app wants to write clipboard */ {})
renderer.onResize(({ cols, rows }) => socket.send(JSON.stringify({ type: 'resize', cols, rows })))
```

`ProgressPayload`:

```ts
interface ProgressPayload {
  state: 'none' | 'normal' | 'error' | 'indeterminate' | 'paused'
  value: number // 0..100
  raw: string
}
```

## Workerized Mode

Set `workerized: true` to run the WASM core in a Web Worker. The renderer
keeps a cached snapshot on the main thread and proxies all mutating calls
through `postMessage`; `snapshot()`, `search()`, and `encodeMouseReport()`
answer synchronously from the cache.

```ts
const renderer = createTermyRenderer(host, {
  termy,        // still required — main thread loads WASM once
  workerized: true,
})
```

Tradeoffs:

- Parser cost moves off the main thread — typing latency improves under
  heavy output.
- `search()` returns the previous result while a new async query is
  re-issued. Same for `encodeMouseReport` (the bridge ports the Legacy / SGR
  encoders to JS for synchronous answers).
- `responses` from `write()` arrive one cycle late.
- The worker chunk ships as `dist/worker.js` and is fetched lazily via
  `new URL('./worker.js', import.meta.url)` — Vite / webpack 5+ / Parcel /
  esbuild auto-handle the URL rewrite.

## Bell

`bellSound: 'visual'` paints a 150 ms full-frame foreground-color flash on
each `BEL`. `bellSound: 'audio'` plays an 80 ms 440 Hz sine via WebAudio
(lazy-init; silently degrades if the browser blocks autoplay). Default `none`
ignores bells. Subscribe to `onBell()` to drive your own UI alongside.

## Config and Themes

```ts
const config = termy.renderConfigFromContents(`
theme = nord
font_family = "JetBrains Mono"
font_size = 14
`)

// pass directly
const renderer = createTermyRenderer(host, { termy, renderConfig: config })

// or just hand the text — the renderer resolves + applies
const renderer = createTermyRenderer(host, {
  termy,
  configContents: 'theme = nord\nfont_size = 14\n',
})
```

`renderConfigFromContents` returns diagnostics alongside the resolved values
so you can surface user-editable config errors.

## Headless / SSR

```ts
const term = createTermyRenderer(null, {
  termy,
  backend: 'headless',
  cols: 80,
  rows: 24,
})

term.write('hello\r\n')
const frame = term.snapshot()
const ansiBytes = term.serialize()    // round-trip the visible buffer
```

Useful for: golden-frame fixtures fed from byte streams, AI agents replaying
PTY transcripts, screenshot services, server-side terminal previews.

## Migration: from xterm.js to first-party rendering

The legacy `attachTermyToXterm` bridge double-parses every byte — once via
`xterm.write()` for rendering and again via `core.feed()` to keep libtermy's
snapshot in sync. The first-party `createTermyRenderer` does both in one
pass.

```ts
// Before — xterm adapter (deprecated, double-parses input)
import { Terminal } from '@xterm/xterm'
import { attachTermyToXterm, loadTermy } from 'libtermy.js'

const xterm = new Terminal()
xterm.open(document.querySelector('#terminal')!)
const termy = await loadTermy()
const bridge = attachTermyToXterm(xterm, {
  termy,
  onInput(data) { socket.send(data) },
})
socket.addEventListener('message', (event) => bridge.write(event.data))
```

```ts
// After — first-party renderer
import { loadTermy, createTermyRenderer } from 'libtermy.js'

const termy = await loadTermy()
const renderer = createTermyRenderer(document.querySelector('#terminal')!, {
  termy,
})
renderer.onInput((data) => socket.send(data))
socket.addEventListener('message', (event) => renderer.write(event.data))
```

Key differences:

- `renderer.onInput` yields `Uint8Array`, not `string`. Use a `TextDecoder`
  if your transport requires UTF-8 strings.
- The renderer exposes `onResize`, `onSelectionChange`, `onLink`, `onTitle`,
  `onWorkingDirectory`, `onProgress`, `onBell`, `onClipboardStore`, `fit`,
  `searchAndHighlight`, `serialize`, `getSelection`, `copy`, `paste`,
  `scrollLines`, `scrollToBottom` directly.
- Drop the `@xterm/xterm` dependency and any xterm theme wiring; pass
  `configContents` or `renderConfig` to `createTermyRenderer` instead.

The xterm bridge emits a one-time `console.warn` on first use. Pass
`{ silenceDeprecation: true }` to silence it in tests.

## Bundlers

The published package includes:

- `dist/index.js` — main bundle (embedded WASM bytes)
- `dist/worker.js` — Web Worker entry (only fetched when `workerized: true`)
- `dist/index.d.ts` — types
- `wasm/termy_wasm.{js,bg.wasm}` — raw wasm-bindgen output for advanced hosts

ESM only. Zero-config under Vite, Bun, webpack 5+, Parcel, esbuild. If you
override `wasmUrl`, serve `.wasm` files with `application/wasm`.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| WASM 404 in production | Use embedded default (no `wasmUrl`), or pass an explicit `wasmUrl` matching your framework's public asset path. |
| Terminal renders but shell hangs on prompts | `renderer.onInput` forwards device-status responses automatically — make sure your transport actually writes them back to the PTY. |
| Input echoed twice | Don't construct two renderers on the same host. Call `dispose()` on unmount. |
| Bracketed paste markers visible in output | The TUI doesn't enable `?2004h`. Paste is wrapped only when the mode is active. |
| Colors mismatched | Pass `configContents` or `renderConfig`; the canvas2d default theme is a built-in fallback. |
| Workerized snapshot lag | The bridge dispatches a `termy:snapshot` `CustomEvent` on the host; both backends listen and call `scheduleRender()`. If your host element is replaced, re-create the renderer. |
| Wide CJK glyph spills | Confirm the WASM build is `0.2.0+` — wide-char support requires the `unicode-width` dep. |
